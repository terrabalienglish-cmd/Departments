Section 01

The Cohort Cycle

Every cohort follows the same structure. Eight phases. Each phase has a clear owner, a clear output, and a clear handoff to the next. The Program Director manages the full timeline. No phase starts without the previous one completed.

Phase 1 · Weeks -6 to -4 Before Cohort Start
Pre-Cohort Setup
Define cohort dates, theme, and capacity. Set KPIs. Brief Marketing on application campaign. Brief Logistics on kick-off event requirements. Open mentor recruitment. Design mentee application form.
Phase 2 · Weeks -4 to -2 Before Cohort Start
Recruitment & Selection
Mentor applications open and close. Screen and approve mentors. Mentee applications open and close. Screen and accept mentees. Send acceptance and rejection communications. All confirmed participants notified.
Phase 3 · Week -1 Before Cohort Start
Onboarding
Run mentor orientation session. Run mentee orientation (Week 0). Collect signed commitment agreements. Assign mentors to groups. Set up WhatsApp cohort groups. Set up cohort dashboard in Notion. Confirm session schedule with all participants.
Phase 4 · Weeks 1–4
Cohort Delivery — First Half
Weekly mentor group sessions run. Weekly mentee-only sessions run. Attendance tracked post each session by Compliance. Weekly tasks assigned and followed up. Check-in messages sent. Engagement monitored.
Phase 5 · End of Week 4
Mid-Cohort Review
Program Director reviews attendance data. Individual check-ins with at-risk mentees. Mentor performance check and escalation if needed. Session quality assessment. Adjustments made for second half. Report submitted to COO.
Phase 6 · Weeks 5–8
Cohort Delivery — Second Half
Sessions continue. Completion tracking active. Graduation list being prepared. Outstanding tasks and absences resolved. Alumni status confirmed for qualifying mentees.
Phase 7 · End of Week 8
Graduation
Graduation list finalised and approved. Graduation ceremony run at workshop event. Certificates issued. Alumni tagged in system. Invite extended to volunteer pool. Graduate stories shared with Marketing for content.
Phase 8 · Week After Graduation
Retrospective & Reset
Post-cohort retrospective with mentors and Program Director. Document learnings. Update playbook. Submit retrospective report to COO. Begin setup for next cohort.
Section 02

Detailed Processes

Each process below is a repeatable workflow. Follow it in order. Every step includes who owns it, what tool is used, and what triggers the next step.

Process 01 — Mentor Recruitment & Selection
01
Set mentor criteria for this cohort
Define the minimum profile: industry, years of experience, what stage of founder they should work with. Write this in Notion before opening applications.
Trigger: 6 weeks before cohort start
Program DirectorNotion
02
Brief Marketing on mentor recruitment campaign
Send Marketing a written brief: what kind of mentor you're looking for, why Terra mentors, the application link, and the deadline. Marketing does not decide the message — they execute it.
Trigger: Brief sent within 48h of criteria being set
Program Director → MarketingNotion brief
03
Open mentor application form
Activate the Google Form. Share link with Marketing. Form collects: name, background, business type, why they want to mentor, availability, session format preference.
Trigger: Marketing brief approved
Program DirectorGoogle Form
04
Review applications against criteria
Score each application using the standard rubric in Notion. Flag borderline cases for a second read. The decision is made by Program Director — not committee. Aim for 3–5 mentors per cohort maximum.
Trigger: Application window closes (5 days before mentee applications open)
Program DirectorNotion + Google Sheet
05
Send acceptance and rejection communications
Accepted: personal email with welcome, what to expect, and orientation date. Rejected: brief, kind, specific. Never ghosted. Every applicant gets a response within 48h of decision.
Trigger: Selection complete
Program DirectorGmail
06
Run mentor orientation session
60-minute session. Covers: Terra's values, what mentors are and are not responsible for, session format, attendance expectations, how to handle difficult mentee situations, escalation path. Every mentor leaves knowing exactly what's expected.
Trigger: 1 week before cohort starts
Program DirectorIn-person or online
07
Collect signed mentor expectations document
Every mentor signs the expectations document before being confirmed. No signature, no confirmation. Stored in Google Drive under Cohort folder.
Trigger: Post-orientation, before cohort start
Program DirectorGoogle Drive
Process 02 — Mentee Application & Selection
01
Set mentee cohort capacity
Decide max number of mentees based on mentor count. Rule: maximum 5 mentees per mentor. Never exceed this. Quality over scale.
Trigger: Mentor selection complete
Program DirectorNotion
02
Brief Marketing on mentee application campaign
Written brief: who should apply, what they get from the program, the ICP profile, application deadline, and any cohort-specific angle. This feeds all content and the kick-off event CTA.
Trigger: Capacity confirmed. Brief sent minimum 3 weeks before applications open.
Program Director → MarketingNotion brief
03
Open mentee application form
Activate Google Form. Form collects: name, business type, stage, specific challenge they want help with, what they commit to, and agreement to attendance policy. The commitment section is non-skippable.
Trigger: Quarterly kick-off event (primary launch point)
Program DirectorGoogle Form
04
Review applications against cohort ICP
Program Director reviews all applications. Prioritise: clarity of challenge, commitment demonstrated in application language, stage match with available mentors. Use scoring rubric in Notion.
Trigger: Application window closes
Program DirectorNotion + Google Sheet
05
Send acceptance communications
Accepted: personal email with welcome, orientation details, what to prepare, and commitment reminder. Waitlisted: honest, kind, specific timeline. Declined: always responded to. Never ghosted.
Trigger: Within 48h of selection complete
Program DirectorGmail template
06
Assign mentees to mentor groups
Match based on: business stage, challenge type, industry where relevant. Document assignments in Notion cohort database. Notify mentors of their group composition before orientation.
Trigger: All acceptances confirmed
Program DirectorNotion
Process 03 — Weekly Session Cadence
01
Send weekly session reminder (Monday morning)
WhatsApp cohort group: time, location/link, what to bring. Keep it short. Keep it consistent. Same format every week so mentees stop reading it and just know.
Trigger: Every Monday of cohort
Program Director or VolunteerWhatsApp
02
Run mentor group session
Mentor leads the session using their own format. Program Director does not interfere. Session structure: check-in (10 min), focus work (40 min), takeaways and task assignment (10 min). Attendance marked by Volunteer in real time.
Trigger: Scheduled session day
Mentor (facilitated) + Volunteer (attendance)In-person
03
Submit attendance to Compliance within 2h
Volunteer submits attendance data to Compliance Lead. Format: cohort name, session number, date, present/absent list. No exceptions on timing — the 2h rule protects the whole reporting chain.
Trigger: Session ends
Volunteer → ComplianceGoogle Sheet
04
Run mentee-only session (same week)
Program Director facilitates. No mentors present. This is where mentees share what they're not ready to share with mentors yet — blockers, confidence issues, group dynamics. What's said here stays here.
Trigger: 2–3 days after mentor session
Program DirectorIn-person or online
05
Assign weekly task and follow up
One clear task per mentee, agreed in session. Posted to WhatsApp cohort group same day. Check-in message sent mid-week. No task completion = flag in Notion tracker, not a public message.
Trigger: Post-mentor session
Mentor assigns, Program Director tracksNotion tracker
06
Flag issues before next session
If a mentee misses a session or fails to complete tasks 2 weeks in a row, Program Director reaches out privately before the next session — not after. Early intervention prevents dropout.
Trigger: Compliance alert or personal observation
Program DirectorGmail or WhatsApp direct
Process 04 — Mid-Cohort Review
01
Pull attendance and task completion data from Compliance
Request updated dashboard from Compliance Lead. Review: who has missed 2+ sessions, who has task completion below 50%, any policy breach flags.
Trigger: End of Week 4
Program DirectorNotion dashboard
02
Conduct 1:1 check-ins with at-risk mentees
Private conversation — not group. Understand the blocker. Offer support. Set expectation. Document the conversation in Notion. This is care, not enforcement.
Trigger: Any mentee with 2+ absences or less than 50% task completion
Program DirectorDirect conversation
03
Check in with every mentor
Brief 1:1 with each mentor. What's working? What's not? Is there a mentee they're concerned about? Is the group dynamic healthy? Any support needed? Document in Notion.
Trigger: End of Week 4
Program DirectorDirect conversation
04
Submit mid-cohort review report to COO
One page. Covers: current completion trajectory, flags, actions taken, any structural adjustments for second half, forecast for graduation numbers. COO is informed, not managing this.
Trigger: Within 3 days of check-ins complete
Program Director → COONotion report
Process 05 — Graduation
01
Compile graduation eligibility list
Pull attendance data. Criteria: attended 6 of 8 sessions minimum, completed at least 60% of weekly tasks, no unresolved policy breach. Program Director makes final call on borderline cases.
Trigger: End of Week 7
Program Director + ComplianceGoogle Sheet + Notion
02
Notify graduates in advance
Private message to each graduate: congratulations, graduation ceremony details, what graduation means for their Terra membership. Also notify non-completers privately and respectfully — with door open for next cohort.
Trigger: 5 days before graduation ceremony
Program DirectorGmail template
03
Brief Marketing on graduation content
Provide graduate names (with permission), key stories from the cohort, any transformation moments. Marketing creates recognition content. Nothing goes out without graduate approval.
Trigger: Graduation list finalised
Program Director → MarketingNotion brief
04
Run graduation ceremony at workshop event
10 minutes within a regular workshop event. Public recognition of each graduate by name and what they worked on. Certificates presented. Invitation extended to alumni community and volunteer pathway.
Trigger: Final week of cohort, coordinated with Workshop Director
Program Director (MC)Workshop event
05
Tag graduates as alumni in system
Compliance tags all graduates in the master dashboard. Program Director updates Notion cohort database. Graduates added to Terra Alumni WhatsApp group. Certificate PDFs stored in Drive.
Trigger: Same day as ceremony
Compliance + Program DirectorGoogle Drive + Sheet
Process 06 — Post-Cohort Retrospective
01
Collect mentor feedback (async)
Send mentors a short Google Form within 3 days of graduation. Covers: what worked, what didn't, support they needed but didn't get, would they mentor again. Anonymous option available.
Trigger: 3 days post-graduation
Program DirectorGoogle Form
02
Collect mentee feedback (async)
Separate Google Form sent to all mentees — completers and non-completers. Non-completers often give the most useful feedback. Covers: expectations vs reality, where they got stuck, what would have kept them engaged.
Trigger: Same day as mentor form
Program DirectorGoogle Form
03
Run retrospective session with mentor team
60-minute session. Three questions only: What do we keep? What do we change? What do we add? Program Director facilitates. Decisions documented in Notion before anyone leaves the room.
Trigger: 7 days post-graduation
Program Director + MentorsNotion (live doc)
04
Update playbook with learnings
Every decision from the retrospective gets reflected in this document. The playbook is alive. If a process failed, the process changes. Not the people.
Trigger: Within 5 days of retrospective session
Program DirectorNotion (this document)
05
Submit retrospective report to COO
Final cohort numbers, completion rate vs target, key learnings, changes made for next cycle, readiness for next cohort. One page. Submitted before next cohort planning begins.
Trigger: Playbook updated
Program Director → COONotion report

Three tools. Everything in its place. Nothing stored in personal devices, personal emails, or WhatsApp chats. If it's not in the system, it doesn't exist for Terra.

W
WhatsApp — Community Communication Only
WhatsApp is for communication, not documentation. Anything important that happens in WhatsApp gets logged in Notion same day. No exceptions.
WhatsApp Group Structure — Mentorship
  • Terra Mentorship — Cohort [X] (group per cohort)
    All mentees + mentors + Program Director. Session reminders, task check-ins, announcements. Archived after graduation. WhatsApp auto-delete set to 90 days.
  • Terra Mentors (permanent group)
    All active and alumni mentors. Used for mentor community updates, new cohort invitations, and Terra news. Program Director and COO are admins.
  • Terra Alumni (permanent group)
    All cohort graduates. Community updates, Terra events, occasional opportunities. Not a mentorship group — a continuation of the relationship.
The WhatsApp Rule
WhatsApp is a communication channel, not a record-keeping system. Decisions made in WhatsApp are not official until they are documented in Notion. If someone says "agreed" in a WhatsApp message, that doesn't count. The agreement lives in Notion, or it doesn't exist.

    Operating Rules

    These are non-negotiable. They exist because without them, the quality of the program degrades. If a rule needs to change, change it through the retrospective process — not by exception.

    Non-Negotiable Operating Rules — Mentorship Department
    01The 8-week program length is fixed. No cohort runs shorter. No extensions. If a mentee can't complete, they apply for the next one.
    02Maximum 5 mentees per mentor. If we have 3 mentors and 20 applicants, 5 are waitlisted for next cycle. Quality before volume, always.
    03No mentor starts without a signed expectations document. No mentee starts without a signed commitment agreement. Both filed before Week 1 begins.
    04Attendance is tracked within 2 hours of each session by Compliance. If the data isn't submitted, the Program Director follows up — not waits.
    05At-risk mentees are contacted privately, before the group becomes aware. Public pressure is not how Terra operates.
    06The mentee-only session is confidential. What's said there does not go to mentors. The Program Director holds this boundary without exception.
    07Every mentor gets a mid-cohort check-in. Not at graduation. Not in the retrospective. At Week 4 — while there's still time to help.
    08The graduation list is approved, not assumed. Program Director signs off on every name. Data from Compliance is the deciding evidence, not memory or impression.
    09The retrospective happens every cycle. No exceptions for "we know what went wrong already." The discipline of the process is the point.
    10Nothing important lives in WhatsApp. If it was said in WhatsApp, it doesn't count unless it's in Notion.
    11The Program Director trains 2 successors, always. One departure cannot interrupt a cohort in progress.
    12Every applicant gets a response. No ghosting. Terra's reputation is built in how it treats people who didn't get in, not just those who did.
    Section 06

    Quarterly Kick-Off Events

    Kick-off events are the primary application funnel for the mentorship program. They are owned by the Mentorship department, executed by Logistics, and promoted by Marketing. The Program Director drives the strategy. Everyone else executes.

    Process — Kick-Off Event End to End
    01
    Define kick-off event strategy and goals
    What is this event for? Primary goal is always: open mentorship applications and fill the cohort. Secondary goal: celebrate the graduating cohort from the previous cycle. Define the theme, the vibe, the CTA, and the target audience profile.
    Trigger: 6 weeks before event date
    Program DirectorNotion
    02
    Send event brief to Marketing
    Written brief. Covers: event name, date, location, theme, audience, the story to tell, the CTA, any graduate stories to feature, and content needed (promotional reel, graphics, announcements).
    Trigger: 5 weeks before event
    Program Director → MarketingNotion brief
    03
    Send event brief to Logistics
    Logistics brief covers: expected attendance, venue requirements, AV needs, registration setup, crew needed, run-of-show outline, sponsor requirements. Minimum 3 weeks ahead — not 1 week.
    Trigger: 4 weeks before event
    Program Director → LogisticsNotion brief
    04
    Review and approve all promotional content
    Every piece of content Marketing creates for this event is reviewed by Program Director before publishing. Not for design — for message accuracy and alignment with Terra's values and CTA.
    Trigger: As Marketing produces content (ongoing from brief)
    Program Director approvesNotion + WhatsApp
    05
    Open mentee application form on event day
    Applications open at the event — not before. The event is the launch. QR code on screen and at registration desk. Volunteers facilitate sign-ups during the event.
    Trigger: Event day
    Program Director + ComplianceGoogle Form
    06
    MC the mentorship section of kick-off
    Program Director speaks for 10–15 minutes maximum. Content: what the program is, who it's for, a story from the graduating cohort, the CTA. This is not a presentation — it's an invitation. Prepared, not improvised.
    Trigger: During event (coordinated with Logistics run-of-show)
    Program DirectorLive
    07
    Follow up with event attendees within 48h
    Compliance provides list of everyone who registered at the event. Program Director sends personalised follow-up to anyone who expressed interest in mentorship. Not a mass email — targeted outreach.
    Trigger: Within 48h of event
    Program DirectorGmail + Notion
    08
    Post kick-off debrief with Marketing and Logistics
    30-minute debrief within 3 days. Three questions: what went well, what broke, what changes for next quarter. Decisions logged in Notion kick-off retrospective. No debrief = same mistakes next quarter.
    Trigger: Within 3 days of event
    Program Director + Marketing + LogisticsNotion retro doc
    Kick-Off Event Brief Template — Fields
    • ▸Event name and quarter
    • ▸Date, time, location
    • ▸Primary goal + secondary goal
    • ▸Target audience profile and expected attendance
    • ▸Event theme and story angle
    • ▸The CTA (exact wording)
    • ▸Graduate stories to feature (with permission)
    • ▸Content needed from Marketing (list)
    • ▸Logistics requirements (space, AV, crew, sponsor requirements)
    • ▸Application form QR code
    • ▸Run-of-show outline
    Kick-Off Success Metrics
    • Application form opened at the event
      Not after. Not a week later. The momentum is live at the event.
    • 30%+ of attendees enter the funnel
      Funnel = registered interest or submitted application.
    • Cohort fills within 2 weeks of event
      If the cohort isn't full within 2 weeks, debrief to understand why before next quarter.
    • Debrief done within 3 days
      Decisions documented. Not discussed and forgotten.
